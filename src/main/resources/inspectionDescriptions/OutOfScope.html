Reports Concord YAML files that do not belong to any project scope.

In the Concord plugin, a Scope is defined by a concord.yaml root file. Only files included in a scope (via resources patterns or by being a root file themselves) benefit from full IDE support, such as cross-file navigation, completion, and validation.

If a file is "out of scope," the IDE cannot safely resolve flow calls or check for duplicates across files.

To fix this:

  • Create a concord.yaml file in the project root or in the current directory to start a new scope.
  • Adjust the resources patterns in an existing concord.yaml file (e.g., "glob:flows/**/*.concord.yaml") to include this file.

This inspection is applied only to Concord YAML files.
