Reports issues in flow documentation blocks.

Flow documentation uses special ## markers to document flow parameters with types. This inspection checks for:

  • Duplicate parameter names in in: or out: sections
  • Unknown parameter types (valid types: string, boolean, int, integer, number, object, any and their array variants)
  • Orphaned documentation blocks that are not followed by a flow definition

flows:
  ##
  # in:
  #   bucket: string, mandatory
  #   bucket: int, optional   <-- duplicate parameter
  #   count: foo              <-- unknown type
  ##
  processS3:
    - task: s3

This inspection is applied only to Concord YAML files.
