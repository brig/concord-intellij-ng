Reports a warning when the same flow name is defined more than once within the same Concord scope. A Concord scope is determined by the nearest concord.yaml root and its resources configuration (the set of files included into the process/project scope).

Duplicate flow names create ambiguity for navigation (Ctrl+B), completion, reference resolving, and flow execution, because the IDE may pick a different definition than you expect.

What is checked

  • Only flow definitions are inspected (a YAML key under flows: that resolves to a Concord step/flow meta type).
  • The inspection searches for other files in the same Concord scope that also define the same flow name.
  • If at least one duplicate is found, a warning is reported on the flow name key.

Example

Problematic:

# project-a/concord/utils.concord.yaml
flows:
  utilsFlow:
    - log: "Utils A"

# project-a/concord/more-utils.concord.yaml
flows:
  utilsFlow:
    - log: "Utils B"

Result: the second (and subsequent) definition(s) of utilsFlow in the same scope will be highlighted.

How to fix

  • Rename one of the flows so that names are unique within the scope.
  • Alternatively, move one of the definitions outside the scope (adjust resources patterns in concord.yaml).

Notes

  • The message includes a relative path to one of the duplicate definitions found in the scope.
  • No quick-fix is provided (renaming typically requires choosing the intended target and updating call sites).
