configuration

Type: object

Profile-specific configuration. Overrides root configuration values when this profile is active.

Keys:

  • arguments (object) — default process arguments, available as top-level variables in flows
  • debug (boolean) — enables debug logging for the process
  • dependencies (string[]) — additional Maven dependencies to include at runtime
  • entryPoint (string) — name of the starting flow (default: 'default')
  • events (object) — configures process event recording and truncation to prevent large or sensitive values from being stored in events
  • exclusive (object) — ensures only one process runs per project + group. Conflict behaviour is controlled by mode.
  • extraDependencies (string[]) — extra dependencies appended to the main dependency list
  • meta (object) — process metadata map exposed by flows; values are updated after each step and can be retrieved via the API or shown in the Concord Console process list. Supports nested keys (e.g. nested.value) and form fields (e.g. myForm.myValue)
  • out (string[]) — list of variables to return as process output
  • parallelLoopParallelism (integer) — maximum number of parallel threads for parallel loops
  • processTimeout (string) — maximum process execution time (Format: ISO 8601 duration, e.g. PT1H)
  • requirements (object) — execution requirements for agent selection and resource constraints
  • runtime (runtime) — runtime version for the process (e.g. concord-v2)
  • suspendTimeout (string) — maximum time a process can stay in SUSPENDED state (Format: ISO 8601 duration)
  • template (string) — name of the template to apply to this process
