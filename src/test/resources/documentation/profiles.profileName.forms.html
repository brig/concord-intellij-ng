forms

Type: object

Declares web-based forms used by flows to collect user input. Each form is defined under a name; the name is used to reference the form from flows and as the object that stores submitted field values.

Form definitions can be reused within the same process, and fields can also be defined dynamically at runtime.

Note: form names may contain only alphanumerics, whitespaces, underscores (_) signs

Example:

forms:
  myForm:
    - myValue: { type: "string", label: "Value" }

flows:
  default:
    - form: myForm
