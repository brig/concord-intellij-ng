What's New in Concord IntelliJ 0.22.0

May 1, 2026 · 3 min read
Brig
Concord IntelliJ Plugin Maintainer

The 0.22.0 release adds schema support for Configuration Validation Settings, extends EL completion with Set Step Self-Reference, and makes Code Folding smarter by using a step's name as the folded placeholder.

Configuration Validation Settings​

The plugin now understands Concord's configuration.validation block — both at the top level and inside profiles. Completion suggests the taskCalls.in and taskCalls.out keys, and the value is restricted to disabled, warn, or fail. Unknown modes are flagged as errors, and hover documentation explains each option.

Named Step Folding​

When a flow step has a name field, the folded placeholder now displays that name instead of the step's identity value. This makes folded flows readable at a glance when steps are descriptively named — - task: Upload artifacts is far more useful than - task: s3 in a long deploy.

flows:
  deploy:
    - task: s3              # folds to: - task: Upload artifacts
      name: "Upload artifacts"
      in:
        bucket: my-bucket
        src: ./build/

Steps without a name keep the previous behavior — folding to the identity value (e.g. - task: s3).

Set Step Self-Reference​

Variables declared earlier inside a set step are now visible in EL completion within the same step — useful when one assignment derives from another. The variable being currently edited is excluded from its own suggestions, so you won't see self-referencing noise.

flows:
  main:
    - set:
        bucket: "my-bucket"
        path: "${bucket}/data"   # `bucket` is suggested here

Other Improvements​

  • Commented-Out Steps Highlighting: Concord keywords inside YAML # comment lines are no longer colored as live syntax — commented-out steps now look like comments instead of mixed code/comment.
  • v1 Task Filtering: Task name completion now skips legacy v1 task classes from dependency JARs, so only v2 tasks (those implementing com.walmartlabs.concord.runtime.v2.sdk.Task) appear in the completion popup.
  • Static Method Completion: Public static methods on tasks are now included in EL task method completion (${myTask.}), alongside instance methods.
